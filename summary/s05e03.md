# s05e03 — Rozwój funkcjonalności

## Stabilność fundamentów vs dynamika warstw wyższych

Architektura transformerów, autoregresja, tokenizacja, limity kontekstu, halucynacje — stabilne od 3 lat. Postęp to skalowanie + syntetyczne dane treningowe + reasoning w inferencji, nie przełom rangi następcy transformerów. Zrozumienie mechanik (np. [What ChatGPT is doing](https://writings.stephenwolfram.com/2023/02/what-is-chatgpt-doing-and-why-does-it-work/)) to inwestycja długoterminowa — materiały z pierwszej edycji AI Devs nadal aktualne na poziomie zasad.

Warstwy wyższe (modele, techniki, narzędzia, API, otoczenie agentowe) — bardzo dynamiczne. Konsekwencja projektowa:
- Logika oparta na fundamentalnych mechanikach → mniejsze ryzyko deprecjacji
- Logika oparta na konkretnych API/frameworkach → planuj wymienność, multi-provider

## Dwuznaczność architektury AI

**Uproszczenie:** oddelegowanie logiki do modelu. Agentic RAG < klasyczny RAG pod względem ilości kodu — agent z serią narzędzi zastępuje wieloetapowe pipeline'y.

**Komplikacja:** środowisko agenta — sandboxy, multimodalność, długie horyzonty, integracja z systemem, bezpieczeństwo.

Metafora: programowanie klasyczne = budowanie elementów linii produkcyjnej. Programowanie generatywne = budowanie fabryki zdolnej do kształtowania tych elementów.

Agentic jako **domyślne podejście** — deterministyczną logikę wybiera się tylko przy istotnym powodzie. Domyślnie zakładaj multimodalność.

## Strategia migracji modeli i API

**Responses API** przyjęło się jako standard branżowy w ~4 miesiące (inni providerzy, SDK, frameworki). **Assistants API** — deprecated po 15 miesiącach (nie szeroko wykorzystane). Różnica: Responses API rozszerza uniwersalny format wiadomości o function calling i structured output, nie uzależnia od provider-specific abstrakcji (threads, runs).

Zasady migracji:
- Mniejsze wersje (mini/nano) mogą sprawiać wrażenie większych możliwości — wymagają dokładniejszej ewaluacji
- Nowsze modele zmiękczają wcześniej rekomendowane praktyki — np. agresywny ton (CRITICAL, MUST) w Opus 4.5 przestał być rekomendowany przez [Anthropic](https://platform.claude.com/docs/en/build-with-claude/prompt-engineering/claude-prompting-best-practices)
- Migracja to nie zmiana identyfikatora — w systemach agentowych wymaga reewaluacji instrukcji, uproszczenia promptów lub odwrotnie — zwiększenia złożoności
- Sprawdzaj nie tylko stabilność, ale i czy migracja pozwala uprościć instrukcje lub robić rzeczy wcześniej niemożliwe
- Obserwuj modele Open Source przez OpenRouter — potencjalnie nieporównywalnie szybsze i tańsze

Nowości warte uwagi: sam sprawdzaj, zachowaj dystans do szumu marketingowego, daj drugą szansę gdy temat wraca wielokrotnie. Obserwuj prace badawcze providerów i [Model Spec](https://model-spec.openai.com/2025-12-18.html) jako wskaźnik kierunku.

## Rozwój agentów przez narzędzia, nie kod

Agent z dostępem do systemu plików + pamięć potrafi programować aplikacje (np. dwie mini-gry z SQLite z jednego zdania instrukcji). Rozszerzenia = dodanie serwerów MCP, zero zmian w logice agenta:

| Rozszerzenie | Zysk | Koszt implementacji |
|---|---|---|
| Lepszy model | Natychmiastowa poprawa jakości | Zmiana stringa |
| Terminal (bash) | Build, test, git, deploy | Nowy MCP server |
| Przeglądarka | Interakcja z web UI | Nowy MCP server |
| Gmail/Calendar/Todoist | Automatyzacja workflow | Nowy MCP server |

Architektura agenta agnostyczna wobec narzędzi — dynamiczne odkrywanie (`listTools()`), transformacja schematów MCP → function calling, zero hardcodowania.

## Automatyczna optymalizacja promptów

DSPy (Python, [dspy.ai](https://dspy.ai/)) i Ax (TypeScript, [axllm.dev](https://axllm.dev/)) realizują automatyczną optymalizację produkcyjnie. Dwa mechanizmy:

### Optymalizacja instrukcji — hill-climbing

Generuj kandydatów → ewaluuj → zachowaj lepszego. Architektura 3-rolowa:

| Rola | Model | Reasoning | Uzasadnienie |
|---|---|---|---|
| execution | gpt-4.1-mini | none | Tłumaczy prompt na JSON — szybki, tani, wywoływany najczęściej |
| judge | gpt-5.4 | high | Precyzyjna ocena semantycznego podobieństwa |
| improver | gpt-5.4 | high | Diagnoza typów błędów, planowanie zmiany |

Rozdzielenie ról = niezależna optymalizacja kosztu vs jakości.

**Best-of-N kandydatów** — K kandydatów równolegle z kierunkowymi strategiami (balanced, coverage, simplify, boundary, salience), ocena wszystkich, wybór najlepszego. Trzykrotnie zwiększa "keep rate". `Promise.allSettled` zamiast `Promise.all` — pętla kontynuuje gdy kandydat zawiedzie.

**Noise floor** — `delta > max(spread) / 2`. LLM niedeterministyczny — poprawa o 0.01 nie jest istotna statystycznie. Spread z wielu runów; `runs=1` = tryb demo (każda poprawa akceptowana).

**Stuck detection** — 3 odrzucone próby z tą samą operacją → wymuszona zmiana strategii z zabronioną operacją. LLM-y powtarzają te same pomysły ("same-idea cycling").

**Anti-verbosity creep** — największy bias improver LLM: domyślnie dodaje reguły. Wymuszanie "fewer rules = more compliance". Tie-breaker: przy równym score i delcie, krótszy prompt wygrywa.

**Train/verify split** — 2 transkrypcje do optymalizacji, 1 holdout do weryfikacji. Optymalizator nigdy nie widzi przypadków verify — ML 101 zastosowane do prompt engineeringu.

**Metaprompt** — 137 linii instrukcji dla LLM optymalizującego inne prompty. 5 atomowych operacji (REWORD/ADD/REMOVE/MERGE/REORDER), jedna na iterację. Strategia zależna od wyniku: <0.5 napraw fundamenty, 0.5-0.7 dodaj brakującą kategorię, 0.7-0.85 konsoliduj, >0.85 tylko usuń.

**Structured prompt format** — sekcje `<objective>`, `<context>`, `<rules>` (jedna reguła = jeden bullet). Zapobiega degradacji w "wall of text", diffy czytelne.

### LLM-as-judge z konfigurowalną polityką

Sekcje z wagami (suma=1.0), `matchBy` do identyfikacji encji, exact vs semantic matching. Wzorzec "self-consistency scoring" — ocenia nie tylko vs ground truth, ale spójność wewnętrzną predykcji.

## Prompt programming — sygnatury (Ax/DSPy)

Sygnatury eliminują prompt z kodu aplikacji — deklarujesz `input:type -> output:type "opis"`, framework generuje prompt. Przeskok od "prompt engineering" do "prompt programming".

```
emailFrom:string, emailBody:string -> labels:string[], priority:class "high,medium,low"
```

**BootstrapFewShot** — uruchom na training set → zbierz traces z udanych predykcji → zapisz jako dema → użyj jako few-shot. Zero zmian w sygnaturze. `:class` = specjalny typ dla klasyfikacji. Opisy w cudzysłowach stają się instrukcjami. Praktyczna uwaga: BootstrapFewShot nie deduplikuje traces — identyczne wpisy zużywają tokeny.

**Progressive enhancement** — system działa od razu z samą sygnaturą, lepiej z ręcznymi przykładami, najlepiej z zoptymalizowanymi demosami. Każdy poziom użyteczny. `demos.json` jako artefakt kompilacji — usunięcie = powrót do baseline'u.

**Metryka z consistency bonus** — nie tylko accuracy vs ground truth, ale nagroda za wewnętrzną spójność (`needsReply=true` ↔ label `needs-reply`). Jaccard similarity dla częściowej poprawności multi-label. Ważona kompozycja: labeli 50%, priorytet 25%, needsReply 15%, spójność 10%.

## Pułapki wdrożeń produkcyjnych

| Obszar | Ryzyko | Mitigacja |
|---|---|---|
| Rate limit | Unieruchomienie przy skali | OpenRouter, rotacja kluczy |
| Moderacja | Blokada konta za treści userów | Moderation API, filtry wejścia/wyjścia |
| Wydajność | Spowolnienie przy równoległych zapytaniach | Unikanie >100 równoległych calls na usera |
| Koszty | **1-3% userów generuje większość kosztów** | Twarde limity per user, dedykowane klucze, monitorowanie biznesowe |
| Skuteczność | Demo działa, userzy robią coś innego | Obserwacja aktywności, przyciski zamiast czatu gdy możliwe |
| Szybkość | LLM zabija UX | Cache, równoległość, streaming, cząstkowe odpowiedzi |
| Użyteczność | Agent zarządzający todo przez czat < dobry UI | Czat sensowny na skali lub przy złożonych akcjach |

## Przykłady kodu

### 1. Coding Agent — agent loop z MCP i memory (05_03_coding)

**Delta:** kompletny agent w ~100 linii bez frameworków — REPL + bounded loop + MCP + rolling memory.

```typescript
export const runAgent = async ({ session, userMessage, mcp, logger }) => {
  addUserMessage(session, userMessage)
  for (let turn = 1; turn <= MAX_TURNS; turn++) {
    await maybeCompactMemory(openai, session, logger)
    const response = await openai.responses.create({
      model: MODEL,
      instructions: buildInstructions(SYSTEM_PROMPT, session.summary),
      input: session.messages,
      tools: mcp.tools,
      parallel_tool_calls: false,
      store: false,
    })
    for (const item of response.output) {
      if (item.type === 'function_call')
        addToolResult(session, call.call_id, await runToolCall(call, mcp, logger))
    }
    if (pendingToolCalls.length === 0) return finalText
  }
}
```

`parallel_tool_calls: false` — operacje na plikach sekwencyjne. `store: false` — prywatność przez design. `MAX_TURNS = 30` — guard.

### 4. Autoprompt — noise floor + stuck detection (05_03_autoprompt)

**Delta vs coding agent:** agent nie wykonuje zadań — optymalizuje instrukcje. Mechanizmy ochrony przed fałszywym postępem.

```javascript
const noiseFloor = Math.max(candidateSpread, lastEvalSpread) / 2;
// Akceptuj zmianę tylko jeśli delta > noiseFloor

export const detectStuck = (history) => {
  if (history.length < 3) return "";
  const recent = history.slice(-3);
  if (recent.every(e => e.status === "discard")
      && recent.every(e => e.operation === recent[0].operation))
    return `Forbidden this iteration: ${recent[0].operation}`;
};
```

### 6. AX — metryka z Jaccard + consistency bonus (05_03_ax)

**Delta vs 5:** nie tylko accuracy — częściowa poprawność zbiorów + nagroda za spójność wewnętrzną.

```typescript
const classificationMetric = ({ prediction, example }) => {
  const labelScore = jaccard(predLabels, expectedLabels);
  const consistencyBonus = (predNeedsReply === predLabels.includes('needs-reply')) ? 0.1 : 0;
  return Math.min(1, labelScore * 0.5 + priorityScore * 0.25 + replyScore * 0.15 + consistencyBonus);
};
```

## Uzupełnienia

**JSONL logging** — zdarzenia jako JSONL per sesja, każda linia niezależny JSON. Schemat: `user.message`, `turn.start`, `model.response`, `tool.call`, `tool.result`, `memory.compacted`, `turn.done`. Umożliwia rekonstrukcję przebiegu sesji.

**Zero zależności** w autoprompt — `package.json` bez `dependencies`, wyłącznie Node.js stdlib + native fetch (Node 24+) + OpenAI REST. Zaawansowany system AI bez frameworków.

**Context jako prior state** w ewaluacji — test case z `context` ma "prior state" z poprzedniego spotkania. Model musi skopiować stan i aplikować tylko zmiany, nie usuwać wymienionych elementów ("silence = unchanged").
