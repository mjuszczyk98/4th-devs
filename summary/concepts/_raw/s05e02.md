# s05e02 — koncepty

## Semantic Events & Rendering
Architektura event sourcing: zdarzenia SSE jako source of truth, bloki UI jako projekcja materializowana dwiema ścieżkami (pełna O(n) i przyrostowa O(1)). Serwer tłumaczy format providerowy na zunifikowany model zdarzeń. Deduplikacja klient-side przez `eventIdsByMessageId` + `lastSeq`. `store: false` utrzymuje pełną kontrolę kontekstu po stronie serwera.

## Streaming Markdown Rendering
Pipeline pięciu warstw (remend → marked → markdown-it → highlight.js → DOMPurify) dla inkrementalnego renderingu streamowanego Markdown. Podział na `committedSegments` (zamrożone, cachowane) i `liveTail` (re-renderowany). Edge case'y parsowania: zagnieżdżone HTML, niezamknięte bloki `$$`, footnotes. DOMPurify jako niewyłączna ochrona przed XSS z outputu LLM.

## UI Virtualization
Chunkowa wirtualizacja (12 wiadomości na chunk) z `ResizeObserver`, scroll anchoring i asymetryczny pin-to-bottom z histerezą (80px release / 4px reacquire). CSS uzupełniający: `content-visibility: auto`, `overscroll-behavior: contain`, `scrollbar-gutter: stable`. Sprawdzone przy 1500+ wiadomościach.

## Streaming
Batchowanie SSE przez `requestAnimationFrame` — bufor pending, flush raz na klatkę, stabilny 60fps niezależnie od tempa streamingu. Chroni przed render thrashing.

## Voice Agents
Dwie architektury: STT→LLM→TTS (3 modele, pipeline, niższy koszt) vs Realtime (1 multimodalny, najniższa latencja, wyższy koszt). LiveKit Agents: `prewarm` ładuje VAD+MCP raz, `entry` per sesja. WebRTC + JWT krótkożyciowy. Kluczowe zjawiska: VAD, barge-in, detekcja końca wypowiedzi.

## MCP
Dynamiczna integracja z agentem głosowym: JSON Schema → Zod (rekursywna konwersja `toZod`), `mcp.callTool()` owinięte w `llm.tool()`. Prefiksowanie `serverName__toolName` — bezkolizyjne scalenie. `Promise.allSettled` dla graceful degradation (jeden MCP pada, agent działa dalej).

## Agent Loop / Agentic Loop
Server-side streaming loop: query → text + tool_calls → execute tools → results → continue (max 6 kroków). `parallel_tool_calls: true`. Serwer na czystym `node:http`, SSE z abort/close.

## Typewriter UX
Animacja pisania z trybami (off/fast/normal/slow) i gating — blok UI czeka na zakończenie animacji poprzedniego. Ochrona selekcji tekstu: render odkładany gdy użytkownik zaznaczył tekst w live tail.

## Search Decision Ladder
Czteropoziomowe spektrum: kontekst → grep/ripgrep → hybrydowe (FTS + semantyczne) → grafy. Samodzielne bazy wektorowe nierekomendowane. Rozszerzenia SQLite/PostgreSQL wystarczają na większość projektów; dedykowane (Qdrant, Chroma) dopiero przy skali. Rozszerzenia i dedykowane = identyczny proces, nieporównywalnie różna złożoność.

## Mock-first Development
Testowanie UI agenta bez dostępu do LLM: `ScenarioBuilder` generuje deterministyczne sekwencje `StreamEvent`ów z konfigurowalnymi opóźnieniami. `detectScenario()` dopasowuje prompt regexem. Zerowy koszt tokenów przy pełnym pokryciu UI.

## Agent Tooling Ecosystem
Kategoryzacja narzędzi: przeglądarki (agent-browser, browser-use/browserbase), sandbox'y (daytona, e2b), scrapowanie (firecrawl, jina, brave), dokumenty (markitdown), CLI (commander, zx), monitoring (chokidar). Łączenie >1 usługi celowe — różne silniki radzą sobie z różnymi typami stron.
