# s04e01 — koncepty

## Agent Loop / Agentic Loop
Bounded turn-based pętla (max 20 tur) z server-side kontekstem przez `previous_response_id`. Reasoning effort z fallbackiem (xhigh→high). Tool execution w try/catch — błąd nie przerywa loopa, zwraca `function_call_output` z errorem. Paralelne wykonanie tool calli w każdej turze.

## Agent Configuration as Markdown
Template agenta w `<name>.agent.md` z YAML frontmatter (model, tools). Dynamicznie ładowany, wzbogacany o workflows, skills, interpolację `{{date}}`. Wiele agentów z różnymi konfiguracjami w jednym systemie — zachowanie konfigurowalne bez zmiany kodu.

## Context Engineering
Server-side kontekst przez `previous_response_id` w OpenAI Responses API — referencja do poprzedniej odpowiedzi zamiast re-sendu pełnej historii. Redukcja zużycia tokenów z liniowego do stałego per tura.

## Sandbox Code Execution
Code Mode: agent przekazuje inline script LUB script_path do pliku w skills. Runner wstrzykuje helper API (`codemode.vault.{read,write,list,search,move}`, `runtime.exec`), komunikacja sandbox→host przez marker-based IPC (`__CODE_MODE_RESULT__=` / `__CODE_MODE_ERROR__=`). Output parsowany od końca. Wzorzec podobny do Code Mode Anthropic.

## Tool Design
Tool Registry jako `Map<string, Tool>` z `findTool(name)` i `definitions(names?)`. Nowe narzędzie = obiekt z `definition` (OpenAI schema) + `handler`. Rejestr filtruje narzędzia per skill — agent fizycznie nie może użyć narzędzia spoza scope'u aktywnego skilla.

## Skill Plugin System
Katalogi `vault/system/skills/<name>/` z `SKILL.md` (YAML frontmatter + instrukcje). Frontmatter definiuje: `allowed-tools` (scope'owanie narzędzi), `runtime-scripts` (deterministyczne transformacje danych zamiast LLM step-by-step), `user-invocable`, `disable-model-invocation`, `argument-hint`. Komunikat parsowany jako `/skill-name args` → `<metadata>` doklejany do wiadomości + narzędzia ograniczane. Auto-discovery: system skanuje `scripts/` w folderze skilla.

## Sync vs Async Collaboration
Dwie kategorie problemów projektowych. Synchroniczna: interfejs centralny, personalizacja na żywo, szersze uprawnienia, feedback dwukierunkowy. Asynchroniczna: interfejs pominięty, procesy predefiniowane, uprawnienia sandboxowane, raportowanie z minimalnym zaangażowaniem. System hybrydowy: edytor + interfejs poleceń + workflow w plikach + agenci z dedykowanymi narzędziami (brak bezpośredniej współpracy między agentami).

## Parallel Prototyping
AI obniża koszt prototypowania do poziomu równoległego testowania wielu wariantów — z sekwencyjnego na równoległe. Każdy test = odpowiedź na konkretne pytanie decyzyjne. Założenia początkowe błędne — liczy się szybkość korekty, nie doskonały plan.

## AI Deployment Strategy
Brak ustalonych best practices — tempo zmian miesiące, nie lata. Każde wdrożenie = eksperyment. Strategia: buduj system, z którego sam korzystasz — bezpośrednie doświadczenie ograniczeń. MVP celowo minimalny jako punkt odniesienia. Decyzja o niestosowaniu AI równie ważna jak o zastosowaniu.

## Workflow-as-Data
Pliki `.md` w `vault/system/workflows/` z frontmatter `{name, description}`. Ładowane do instrukcji agenta jako obowiązkowe sekcje. Zmiana procesu = zmiana pliku, bez rekompilacji. Agent wykonuje kroki jako część swojego loopa.

## Incremental Sync
Snapshot-based delta synchronization: sygnatura pliku = `size:mtimeMs`. Przesyłane tylko zmienione pliki. Prostsze niż hashowanie zawartości, wystarczające do detekcji zmian. Background loop (700ms) z `syncInFlight` flag zapobiegającym race conditions.

## Provider Abstraction
OpenAI Responses API: `previous_response_id` jako mechanizm kontynuacji sesji, reasoning effort parametr per model z fallbackiem. Konkretne capabilities API wpływające na architekturę agent loop.

## Decision Mapping
Sześć obszarów decyzyjnych mapujących każdą funkcjonalność na uzasadnienie: Użytkownik (programista → złożone narzędzia + sandbox), Treść (ręczna → AI wzbogaca, nie generuje), Format (markdown → HTML przez kod), Integracje (skills + code mode → CLI, MCP, narzędzia natywne), Publikacja (GitHub Pages, statyczny HTML, public-by-design), Dostępność (API + zdalny serwer + Mutagen). Framework stosowalny do każdego wdrożenia AI — wymusza jawne uzasadnienie każdej decyzji projektowej.
