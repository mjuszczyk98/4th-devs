# s01e04 — koncepty

## Multimodality
Głębokie rozwinięcie: problem załączników nie jest adresowany przez żadne API, brak standardu branżowego. LLM widzi zawartość pliku, ale nie adres/referencję — nie może przekazać jej narzędziu ani innemu agentowi. Dotyczy obrazów, audio, wideo, dokumentów. Autorska konwencja: tag `<media src="...">` jako trzeci element wiadomości obok tekstu i pliku, resolver zamienia na Base64/URL tuż przed wywołaniem. Vision/audio modele rozumieją ton, emocje, dźwięki otoczenia, diaryzacja działa bez treningu.

## Workflow vs Agent
Kontynuacja: instrukcja agenta **nie może** zależeć od zestawu danych — tylko od klasy problemów. Workflow i agent nie są wyborem albo-albo: workflow może być narzędziem agenta, agent może delegować do workflow jako jednego z narzędzi. Instrukcja agenta = cel + limity + uniwersalne wzorce rozumowania. Skuteczność agenta <100% to wpisany koszt; weryfikacja człowieka jest częścią modelu.

## System Prompt Design
Nowy wzorzec: **instrukcja oparta na regułach rozumowania, nie krokach procesu**. Przykład `image_recognition`: jak zbierać dowody (obserwowalne cechy), jak dopasowywać (profile = wymagania minimalne, nie wyczerpujące), jak radzić sobie z niejednoznacznością (`unclassified` zamiast zgadywania), jak oceniać kompozyty (każdy obiekt osobno). To realizacja "zależnie od klasy problemów, nie od zestawu danych".

## Augmented Function Calling
Kontynuacja: konfiguracja runtime przez pliki (`style-guide.md`, `template.html`, `template.json`) czytane przez agenta przed pierwszą akcją. Oddziela reguły stylu/szablony od instrukcji systemowej — użytkownik modyfikuje zachowanie agenta edytując pliki markdown/HTML/JSON bez dotykania kodu. Szablon jako master reference — nigdy edytowany bezpośrednio, zawsze klonowany.

## Generative Media Loop (NOWY)
Wzorzec pętli tworzenia mediów: **generuj → analizuj → accept/retry**. Agent nie widzi natywnie outputu narzędzi wizualnych (wygenerowany obraz, wykres, PDF, wynik `execute_code`). Wymaga dedykowanego narzędzia `analyze_image` jako osobnego kroku z ustrukturyzowanym werdyktem. RETRY tylko przy blocking issues; drobne niedoskonałości = ACCEPT — inaczej nieskończona pętla poprawek. Analiza sprawdza prompt adherence, artefakty, anatomię, text rendering, kompozycję.

## JSON Prompt Templates (NOWY)
Strukturyzowanie promptu do generowania mediów jako plik JSON. Zalety: precyzyjna edycja jednej sekcji (`subject`) bez tykania reszty (styl/paleta/oświetlenie/negative prompt), powtarzalność stylu, oszczędność tokenów (agent przekazuje ścieżkę zamiast treści), wersjonowanie (plik per generacja). Workflow: COPY template → EDIT subject → READ full JSON → PASS path. Agent może generować szablon JSON na podstawie obrazów referencyjnych.

## Visual References (NOWY)
Obrazy referencyjne jako tablica `reference_images` sterują kompozycją, pozą, kadrem. Zastosowania: spójność postaci między scenami, in/out-painting. Agent wnioskuje odpowiednią referencję z opisu użytkownika zamiast wymagać nazwy pliku. W wideo: ostatnia klatka segmentu → pierwsza klatka następnego pozwala łączyć krótkie generacje w dłuższe sekwencje.

## Visual Hallucinations (NOWY)
Modele generujące obrazy halucynują analogicznie do tekstowych: ~95% poprawnych detali, ~5% subtelnych glitchy (np. błędny napis na ostatniej klatce). Subtelne błędy trudniejsze do wyłapania niż oczywiste. Weryfikacja ludzka konieczna, zawsze łatwiejsza niż praca od zera.

## Media Cost Control (NOWY)
Wideo jako najdroższa modalność: ~300 tokenów/sekundę, 1h ≈ 1M tokenów. Mechanizmy kontroli: clipping (`start_time`/`end_time`), redukcja FPS, przyspieszenie nagrania. Audio: wybór podejścia (rozdzielone TTS/STT vs multimodal vs real-time) jest trade-offem ceny/jakości/latencji. Analiza wideo/audio bezpośrednio z YouTube URL bez pobierania.

## Document Generation Asymmetry (NOWY)
Generowanie PDF przez HTML → Puppeteer jest proste; przetwarzanie istniejących PDF-ów to osobny poważny problem (dedykowane API providerów, LlamaIndex). Wzorzec generowania: template HTML + style-guide → klon → modyfikacja body → osadzanie wygenerowanych obrazów → konwersja. Iteracyjne poprawki fragmentów bez przerabiania całości.

## Audio UX (NOWY)
Gdy output jest audio, styl agenta musi być dostosowany do medium: bez URL-i, tabel, formatowania markdown. Dotyczy zarówno TTS jak i interfejsów głosowych real-time. Kontrola stylu TTS przez naturalny język wstawiany w tekst ("Say cheerfully:", "In a whisper:"). Multi-speaker przez przypisywanie głosów do rozmówców.
