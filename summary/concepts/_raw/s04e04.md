# s04e04 — koncepty

## Knowledge Base Design
Lekcja implementuje podejście "budowanej" bazy wiedzy z pięcioma strefami odpowiedzialności (Me/World/Craft/Ops/System), jawnym podziałem właściciela (człowiek vs agent) i konstytucją vaultu w `mind.md`. Kluczowe: baza wiedzy = kod źródłowy agenta, pytanie "co agent musi wiedzieć?" nie "jak zbudować bazę?". Zasada self-sufficient notes — notatki pisane jakby czytelnik nie miał kontekstu. Szablony w `system/templates/` determinują strukturę i lokalizację. Trzecia strefa obok "nasza" i "agentowa": wiedza obecna jako kontekst, nie do automatyzacji.

## Agent Configuration as Markdown
Agent jako plik `system/agents/<name>.md` z gray-matter frontmatter (model, tools) + treść jako system prompt. Loader parsuje i zwraca obiekt konfiguracyjny. Dual-mode prompting: workflow mode (tylko plik fazy) vs standalone mode (templates, index, reguły) — każdy tryb z osobną inicjalizacją unikającą scope creep.

## Workflow-as-Data
Workflow jako katalog w `ops/` z `_info.md` (metadata, depends_on) + pliki fazowe. Instrukcje preskryptywne z listą "Do NOT" — beat goal-oriented. Orchestrator deleguje sekwencyjnie, każda faza = osobny agent z własnym plikiem instrukcji. Prescriptive phase files dają przewidywalność, której vague cele nie dają.

## Multi-agent / Delegation
Delegacja przez tool `delegate` z depth-aware tool availability: przy `MAX_DEPTH` tool jest fizycznie usuwany z listy — constraint enforcement przez architekturę, nie prompt. Agenci delegowani dostają dokładne ścieżki do odczytu/zapisu, nie eksplorują filesystemu — orchestrator jako jedyny z orientacją w strukturze.

## MCP
Dynamiczna rejestracja narzędzi z serwerów MCP przez stdio. Namespace collision prevention przez prefix: `files__fs_read`. Prefix matching (`name.split("__")[0]`) pozwala deklaratywnie wybrać serwer. Scoped access: filesystem rootowany w `./workspace`. Dodanie serwera = linijka w `mcp.json`, zero zmian w kodzie.

## Agent Loop / Agentic Loop
Pętla bounded z conversation accumulation: `[...current, ...calls, ...results]` — model widzi pełną historię wywołań i wyników. `Promise.all` na parallel tool calls. Depth-aware filtracja narzędzi. Brak `previous_response_id` — pełny kontekst re-sendowany co turn.

## Context Engineering
Self-sufficient notes jako zasada jakości kontekstu — cztery typy luk (niejawne referencje, nierozpoznawalne linki, nadpisane wersje, brak powtórzeń) zabijających skuteczność agenta. Dual-mode prompting: agent w trybie workflow nie ładuje szablonów = kontrola szumu. Scatter/gather: wyniki narzędzi do plików, czytanie tylko potrzebnych fragmentów — pełne artykuły nigdy w conversation history.

## Frontmatter Metadata Model
Wieloosiowa kontrola metadanych w YAML frontmatter: publish (cykl publikacji), status (dojrzałość treści), access (dziedziczone z sekcji), attention (sygnał handoffu), tags (płaskie, opisują co nie gdzie). Każda oś niezależna. Uprawnienia dziedziczone per sekcja w `mind.md`, nadpisywane per notatka.

## Scatter/Gather
Wzorzec researchu: wyniki narzędzi zapisywane jako pliki (`outputMode: "file"`), nie inline. Agent czyta tylko potrzebne fragmenty. Kontrola wielkości kontekstu — pełne dokumenty nigdy nie trafiają do conversation history.
