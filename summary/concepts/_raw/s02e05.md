# s02e05 — koncepty

## System Prompt Design
Lekcja rozbija instrukcję na sekcje o odmiennych cyklach życia: Identity (stała persona), Protocol (reguły środowiska), Voice (rozbudowany ton — LLM szybko zapominają), Tools (dynamiczna, schematy opisują się same), Workspace (dane runtime), CTA (zapobiega doklejaniu). Obecność instrukcji nie gwarantuje przestrzegania — projektuj pod pozytywne zaskoczenie, nie dokładne wykonanie.

## Observational Memory
Implementacja Observer/Reflector z trójpoziomową kompresją. Observer ekstrahuje obserwacje z tagami źródłowymi `[user]`/`[assistant]`/`[tool:name]` determinującymi priorytet. Reflector kompresuje w 3 poziomach agresywności — `[user]` nigdy usuwane, `[assistant]` kondensowane pierwsze. Kalibracja estymacji tokenów (`chars/4` + runtime ratio). Guard: jeden observer per HTTP request. Head/tail split z ochroną koherencji (nigdy nie odetnie wyniku narzędzia od wywołania). Generation count śledzi stratę informacyjną.

## Progressive Tool Discovery
Meta-narzędzia (`list_servers`, `list_tools`, `get_tool_schema`, `execute_code`) pozwalają agentowi odkrywać narzędzia MCP w runtime. Rejestr z pełnymi sygnaturami TypeScript — agent ładuje tylko potrzebne do `loadedTools` Map. Oszczędność kontekstu: narzędzia nieładowane nie zajmują okna.

## Sandbox Execution (Code-as-Action)
Agent generuje kod JS zamiast bezpośrednich tool calls, uruchamia w QuickJS z izolacją (128 MB RAM, 5s timeout). Dane jako zmienne w sandboxie — nigdy w kontekście LLM. Asyncified host functions: asynchroniczne MCP narzędzia wywoływane synchronicznie z perspektywy QuickJS. Klucz: wrappery muszą być sync, po eval — `executePendingJobs()` jako flush microtasków.

## Knowledge Categorization
Sześć kategorii wiedzy w systemie wieloagentowym: dokumenty sesji, wiedza publiczna/prywatna/agentów, pamięć podręczna, runtime. Ta sama informacja może należeć do wielu kategorii zależnie od kontekstu — proste zasady organizacji ważniejsze niż złożone systemy kategoryzacji. Czasem wystarczą proste dokumenty zamiast zaawansowanej pamięci długoterminowej.

## Steering via Associations
Identity kieruje uwagę modelu przez celowe słowa o zabarwieniu emocjonalnym ("instynkt", "wyczucie") — mechanizm analogiczny do "nie myśl o niebieskich motylach". Skojarzenia fokusują wzorce zachowania silniej niż dosłowne instrukcje. Halucynacje jako atut: prompting przez skojarzenia na swoją korzyść.

## Security & Sandboxing
Połączenia między narzędziami jako wektor obejścia restrykcji — agent z dostępem do filesystemu i Jiry może przenieść dane do sekcji nieobjętej limitami. Sandbox z code execution jako mitygacja: ograniczony kontakt ze światem zewnętrznym, limit pamięci, timeout, brak filesystem/network.

## Agent Configuration as Markdown
Definicja agenta jako `.agent.md` z gray-matter frontmatter (name, model, tools) + body (system prompt). Zachowanie konfigurowalne bez zmiany kodu. Dynamiczne ładowanie i łatwa iteracja.

## Prompt Iteration with LLM
Kilkanaście iteracji normą nawet przy współpracy z AI. Kontekst do iteracji: możliwości systemu, role innych agentów, preferencje użytkownika, typowe błędy (np. pomijanie wczytywania wspomnień gdy polecenie nie sugeruje tego wprost).

## Nested Delegation
Subagent może sam wzywać innych agentów, nie tylko komunikować z nadrzędnym. Instrukcje muszą uwzględniać reguły przekazywania zadań w głąb hierarchii — wpływa na architekturę całego systemu.
