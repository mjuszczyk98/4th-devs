# s03e05 — koncepty

## Context Engineering
S03e05 wprowadza **temporal grounding** — metadane czasowe wstrzykiwane w każdą wiadomość (`now_iso`, `weekday`, `local_time`, `timezone`, list `recallable`, nudge). Sterują uwagą modelu bez modyfikacji promptu systemowego — bezpieczne dla prompt caching. Nudge w metadanych zachęca do aktywności ("think before you respond; recall when topic shifts") — zmienia się co turę, ale system prompt pozostaje stabilny.

## System Prompt Design
S03e05 wprowadza **warstwową architekturę kognitywną** agenta: tożsamość, zdolności poznawcze, inteligencja emocjonalna, sposoby ekspresji, mechaniki wzmacniające. Instrukcje są zgeneralizowane — opisują postawy i zasady, nie procedury. Kluczowe: wiedza o sobie/otoczeniu pochodzi z zewnątrz (nie bazowa wiedza modelu), agent ma pozwolenie na luźne łączenie faktów, nowe tematy = nowe przestrzenie do odkrycia. Modele (nawet z rozwiniętym PE) słabiej radzą sobie z generalizacją — rola projektanta pozostaje kluczowa.

## Chain-of-Thought / Reasoning Control
S03e05 realizuje CoT jako **explicit tool** (`think`) zamiast prompting technique. Zero I/O — wymusza metarefleksję, wynik wraca jako `function_call_output`. Obecność narzędzia w definicjach zmienia zachowanie modelu nawet bez wywołania (mechanizm nudge). Narzędzie `recall` tworzy drugą warstwę reasoning: model decyduje czy wywołać, scout dostarcza dane.

## Dynamic / Generative UI
S03e05 rozszerza znacznie o **spektrum trzech strategii**: Artifacts (LLM generuje pełny HTML, niska kontrola), Render (JSON spec z katalogu komponentów, server-side rendering, średnia kontrola), MCP Apps (UI predefiniowany, model izolowany od kodu, pełna kontrola). Kluczowe rozróżnienie: widoczność kodu dla modelu (pełna w Artifacts/Render, brak w MCP Apps). Przesunięcie roli projektanta z detali w stronę struktur.

## Multi-agent / Delegation
S03e05 wprowadza wzorzec **scout sub-agent**: separacja wnioskowania (main agent) od eksploracji danych (scout z własną sesją, system promptem i MCP file tools). Scout max 8 tur, czyta tylko to co odpowiada celowi. Sesja persistuje across turns, reset przez `new_session`. Main agent nigdy nie widzi struktury plików bezpośrednio.

## Agent Configuration as Markdown
S03e05 rozszerza o **YAML frontmatter z deklaratywnym resolve'owaniem narzędzi**: `tools: [think, recall]` we frontmatter → tylko wymienione trafiają do modelu. Scout dostaje narzędzia MCP dynamicznie. Gray-matter parsing, agent = plik markdown.

## Observability / Token Tracking
S03e05 dodaje **JSONL trace** per data — pełna trasa decyzji agenta (request, response, tool result z timestampem, modelem, statusem, preview). Label odróżnia sesje `agent` od `scout`. Wzorzec: workspace/traces/{date}.jsonl jako audit trail zachowania.

## Observational Memory
S03e05 wprowadza **workspace jako pamięć kognitywną** z trzema kategoriami analogicznymi do psychologii poznawczej: epizodyczna (snapshoty interakcji), faktyczna (trwałe fakty między sesjami), proceduralna (wyuczone reguły behawioralne). Plus warstwa tożsamości (`profile/user/`, `profile/agent/`) i `system/index.md` jako mapa nawigacyjna dla scouta.

## MCP (Model Context Protocol)
S03e05 rozszerza o **MCP Apps** — serwer MCP wystawia interaktywny HTML jako resource z MIME `text/html;profile=mcp-app`. `registerAppTool`/`registerAppResource` z `@modelcontextprotocol/ext-apps/server`. `visibility: ['app']` scope'uje narzędzia tylko do UI. `structuredContent` zwraca dane dla UI obok `content.text` dla agenta. Dual-mode UI: standalone (REST) i embedded (iframe).

## Tool Design
S03e05 wprowadza **narzędzia jako cognitive scaffolding** — `think` (zero I/O, zmienia zachowanie samą obecnością) i `recall` (delegacja do scout sub-agent). Tool visibility scoping przez `visibility: ["app"]` — ukrywa narzędzia przed modelem, dostępne tylko dla UI. Separation of concerns: conversationalist + retrieval specialist.

## Non-determinism as Feature
Nowy koncept: niedeterminizm LLM jako **źródło wartości**, nie błąd. LLM to "śniąca maszyna" — wszystko jest halucynacją która czasem sprzyja. `temperature`/`top_p` marginalne; zachowanie zdeterminowane przez treść kontekstu. Projektowanie przestrzeni, w której zmienność jest zaletą — agent "świadomy" ma bardzo niskie prawdopodobieństwo powtórzenia odpowiedzi, co jest cechą, nie bugiem.

## Cognitive Architecture
Nowy koncept: przejście od projektowania zachowań do **stwarzania warunków** powstawania zachowań. Referencja: "Cognitive Architectures for Language Agents" (arxiv 2309.02427). Warstwy: tożsamość → zdolności poznawcze → inteligencja emocjonalna → ekspresja → mechaniki wzmacniające. Kontrast z kierunkiem branży (eliminacja halucynacji) — nie wyklucza, wprowadza zmienność. Przesunięcie roli projektanta: detale → struktury.

## Temporal Grounding
Nowy koncept: wstrzykiwanie metadanych czasowych i sytuacyjnych w każdą wiadomość jako fundamentalny building block agentów "świadomych". Agent wie kiedy jest, co może odkryć, dostaje nudge zachęcający do aktywności. Sterowanie uwagą bez modyfikacji system promptu — cache-safe, zmienia się co turę.

## Capability Packs
Nowy koncept: biblioteki ładowane do LLM-generated artifacts jako prelude — model wybiera z manifestu, system resolwuje i wstrzykuje. Przykład: 11 pakietów (Preact, Chart.js, D3, Tailwind, Zod, Day.js, DOMPurify, PapaParse, XLSX, SheetJS, core). Wybór wersji pod kątem capabilities modelu (Tailwind v3 nie v4), nie "najlepszych standardów".

## Theory of Mind in LLMs
Nowy koncept: modele od GPT-4 demonstrują rozwinięte zdolności wnioskowania o stanach mentalnych innych (ref. arxiv 2505.00026). Podstawa mechanik inteligencji emocjonalnej agenta — "czytanie między słowami", odczytywanie intencji, dopasowywanie odpowiedzi. Wzrasta z generacjami modeli.

## Deterministic Artifact Editing
Nowy koncept: edycja LLM-generated content przez search/replace zamiast regeneracji. Model decyduje CO (operacje), system deterministycznie aplikuje (regex). Szybsze, tańsze, przewidywalne. Analogiczne do agentic code editing. `edit_artifact` jako wzorzec dla każdego systemu z iteracyjnym generowaniem.

## External Signal Enrichment
Nowy koncept: sub-agent automatycznie wzbogaca kontekst o sygnały z zewnętrznych API (pogoda, itd.) na podstawie celu/keywordów. Lazy — fetchuje tylko gdy cel wymaga. Wynik wstrzykiwany jako hint. Ogólny wzorzec: agent nie tylko czyta pliki, ale integruje dane z otoczenia.

## Resilient JSON Parsing
Konkretna technika: `extractJsonCandidates()` z 3 strategiami (raw text → fenced code block → first `{` to last `}`), pierwsza która parsuje wygrywa. Reusable pattern dla każdego pipeline'u przetwarzającego output LLM.
