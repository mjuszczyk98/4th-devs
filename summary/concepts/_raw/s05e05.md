# s05e05 — koncepty

## Digital Garden
Cyfrowy ogród — strona www generowana z systemu plików markdown (frontmatter + wikilinks) pełniąca jednocześnie rolę bazy wiedzy agenta, obszaru roboczego (agent czyta/modyfikuje pliki), publikacji (wybrane treści jako www) i organizacji (tagi, wikilinks). Build pipeline: collect → parse → rewrite links → render → search (Pagefind). Auto-build na podstawie fingerprintu SHA-256. Pliki `visibility: private` chronione hasłem. Folder workspace kompatybilny z Obsidian.

## MCP Code Mode
Gdy sandbox aktywny, agent wywołuje narzędzia MCP przez pisanie i wykonywanie kodu, nie przez function calling. Definicje narzędzi nie są wczytywane do kontekstu — agent odkrywa je na żądanie przez file-based IPC bridge (write request JSON → poll response JSON). Konfigurowalne per agent: mali wyspecjalizowani agenci z function calling, duży agent z wieloma integracjami z Code Mode.

## Daily Ops
Wzorzec cyklicznej orkiestracji asynchronicznej: niezależni agenci uruchamiani równolegle, każdy w własnej sesji, każdy zbiera dane z własnych integracji (Calendar, Tasks, Mail, Newsfeed). Wyniki do jednego folderu → agent agregujący → transkrypt + audio → urządzenie mobilne. Uruchamianie przez API z `initialMessage` kierującym do plików opisujących proces. Harmonogram z zewnątrz (cron, GitHub Actions). Brak zależności między agentami = pełen paralelizm.

## Polling Worker Pattern
Proste polling workers z adaptacyjnym opóźnieniem jako alternatywa dla skomplikowanych message queues. Jeśli była praca → delay=0 (natychmiast sprawdź ponownie). Jeśli nie → czekaj `pollIntervalMs`. `wake()` przerywa oczekiwanie. Wystarczające dla single-server SQLite deployment.

## Context Compaction z Boundary Integrity
Mechanizm kompakcji kontekstu zachowujący integralność par tool call/response i pending waits. Granica tail iteracyjnie dostosowywana — kompakcja nie może ciąć w środku interakcji narzędzia, bo model straciłby kontekst trwającej operacji.

## Multi-Lane Event Outbox
Każdy domain event dispatchowany do niezależnych lanes z osobną retry/quarantine logiką: realtime (SSE), projection (read model rebuild), background (side effects), observability (tracing). Duże payloady kompresowane jako sidecars. Failed events po max retries trafiają do quarantine.

## Sandbox Writeback (Explicit Consent)
Sandbox edits domyślnie ephemeral — wymagają jawnego zatwierdzenia przez `commit_sandbox_writeback`. Dwa tryby: bash (inline) i script (JavaScript ES module). Warstwa bezpieczeństwa między "agent wykonuje kod" a "agent modyfikuje pliki" — zmiany nie trwają bez wyraźnej zgody.

## Delegation Handoff Envelope
Strukturalny kontekst przekazania między agentami — child dostaje pełny envelope z informacjami o parent (agentId, revisionId, runId), target (agentId, delegationMode, inputFileIds) i wersji. Delegacja to nie tylko zadanie, to pełny kontekst relacji, wersji agenta i plików wejściowych.

## Block Settling
Wzorzec renderowania streamingowego: bloki podczas streamingu renderują się flat, po zakończeniu następuje faza settling — bloki grupowane w ToolChain (sekwencyjne) i ToolGroup (równoległe) z animacją collapse. Complementarny ze split na committedSegments + liveTail.

## Agent DSL z Rewizjami
Pełny DSL konfiguracji agenta w YAML frontmatter: model, sandbox policy, kernel, memory, garden, subagents, narzędzia (mcpMode, nativeTools). Każda zmiana tworzy nową rewizję z checksum SHA-256 — pełna historia ewolucji konfiguracji. Walidowany przez Zod (schema: agent/v1). Agent nie jest promptem, to pełna konfiguracja runtime.

## Nawyk > Technologia
Techniczne możliwości agentów bez nawyku korzystania = zero wartości. Klucz: dopasowanie kanału komunikacji do istniejących zachowań użytkownika (Discord → powiadomienia, telefon → shortcut, spacer → auto-loading audio). Łączenie nowej aktywności z tym, co już robimy. Samo skonfigurowanie integracji nie wystarczy — musi zostać używane cyklicznie.

## Technologia bez Procesu
System agentowy z pełnym stackiem narzędzi (MCP, sandbox, przeglądarka, delegation) jest bezużyteczny bez zdefiniowanych procesów. Wartość pojawia się dopiero przy spersonalizowanych procedurach, skryptach i cyklicznych wyzwalaczach. Heurystyka: zacznij od jednego powtarzalnego procesu, nie od architektury wszechświata.

## Monorepo ze Współdzielonymi Kontraktami
`packages/contracts` z Zod schemas → branded IDs, zero build step, source of truth dla client/server. Schematy współdzielone między backendem a frontendem — zmiana w jednym miejscu propagowana wszędzie. Typy brandowane (`RunId`, `JobId`) dają compile-time safety.
