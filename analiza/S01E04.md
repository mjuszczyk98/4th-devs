# S01E04 — Multimodalność i załączniki w agentach AI

## Obsługa załączników w API

LLM przesyłany obraz (URL/Base64) trafia do modelu jako dane wizualne — model **nie widzi adresu URL**. Nie może więc przekazać pliku do narzędzia ani innemu agentowi. Problem dotyczy wszystkich modalności: obrazów, audio, wideo, dokumentów.

Rozwiązanie: do wiadomości użytkownika dodajemy **programistycznie** trzeci element obok tekstu i pliku — tag z odnośnikiem (np. `<media src="path/to/file">`). Instrukcja systemowa wyjaśnia agentowi jak się do niego odwoływać. Odnośnik jest potem zamieniany na Base64 lub publiczny URL w zależności od potrzeb narzędzia.

## Instrukcje agenta vs workflow

Workflow ma sztywny, predefiniowany proces — działa gdy dane są statyczne i klasyfikacja prosta. Agent dostaje przestrzeń do działania — instrukcja definiuje:

- **Cel** — co ma osiągnąć
- **Limity** — czego nie powinien robić
- **Uniwersalne wzorce rozumowania** — jak podejmować decyzje

Instrukcja agenta **nie może** zależeć od konkretnego zestawu danych ani prowadzić przez sztywną sekwencję kroków. Może dotyczyć **klasy problemów**.

Workflow może być narzędziem agenta — nie jest to wybór "jedno albo drugie".

## Iteracyjna weryfikacja outputu

Agent nie widzi natywnie wyników narzędzi wizualnych (wygenerowany obraz, wykres, PDF). Potrzebuje **dedykowanego narzędzia do analizy** jako osobnego kroku.

Pętla: **generuj → analizuj → accept/retry**. Werdykt RETRY tylko przy blocking issues (zła poza, złamana kompozycja, nieczytelny tekst). Drobne niedoskonałości = ACCEPT. Bez tego agent wpada w nieskończoną pętlę poprawek.

## Szablony JSON jako prompt engineering

Strukturyzowanie promptów do generowania obrazów jako JSON daje:

- **Precyzyjną kontrolę** — agent edytuje tylko `subject`, reszta (styl, paleta, kompozycja, constraints) zostaje nienaruszona
- **Powtarzalność** — spójność stylu między generacjami
- **Oszczędność tokenów** — agent przekazuje referencję do pliku zamiast przepisywać cały prompt
- **Wersjonowanie** — każda generacja ma własny plik promptu

Workflow szablonu: **kopiuj oryginał → edytuj fragment kopii → przekaż ścieżkę do narzędzia**.

## Grafiki referencyjne

Obrazy referencyjne sterują kompozycją, pozą, kadrem. W połączeniu z JSON prompt dają spójność postaci w różnych scenach. Agent **wnioskuje** odpowiednią referencję z opisu użytkownika ("warrior charging" → `running-pose.png`) zamiast wymagać dokładnej nazwy pliku.

Klatka końcowa jednego fragmentu wideo może być klatką początkową następnego — mechanizm tworzenia dłuższych filmów z modeli ograniczonych do kilku sekund.

## Audio — specyfika modalności

Trzy podejścia: rozdzielone TTS/STT, modele multimodalne, interakcja real-time. Wybór zależy od ceny, jakości i latencji — nie ma jednej najlepszej opcji.

Modele audio rozumieją nie tylko treść, ale ton, emocje, dźwięki otoczenia. Diaryzacja (rozpoznawanie rozmówców) działa bez dedykowanego treningu.

Styl odpowiedzi agenta musi być dostosowany do formatu — bez dyktowania URL-i, tabel, formatowania markdown.

## Wideo — kontrola kosztów

Wideo to najdroższa modalność: ~300 tokenów/sekundę, 1h filmu ~ 1M tokenów. Mechanizmy kontroli:

- **Clipping** (`start_time`, `end_time`) — focus na fragment
- **FPS** — mniej klatek = mniej tokenów
- **Przyspieszenie nagrania** — o ile nie traci się istotnych detali

YouTube jako źródło danych — analiza bezpośrednio przez API bez pobierania.

## Dokumenty PDF

Generowanie (HTML → Puppeteer → PDF) jest proste. Przetwarzanie istniejących PDF-ów — osobny problem wymagający dedykowanych API.

Agent operuje na szablonie HTML + style guide. Klonuje szablon, modyfikuje treść, osadza wygenerowane obrazy, konwertuje. Może iteracyjnie poprawiać fragmenty bez przerabiania całości.

## Halucynacje wizualne

Modele generujące obrazy halucynują tak samo jak tekstowe. Subtelne glitche w 5% detali są trudniejsze do zauważenia niż oczywiste błędy. Skuteczność agenta nie będzie 100% — weryfikacja przez człowieka jest konieczna, ale zawsze łatwiejsza niż wykonanie pracy od zera.

## Przykłady kodu — progresja

### `01_04_image_recognition` — fundament

Wprowadza bazową architekturę powtarzaną we wszystkich kolejnych przykładach:

- **Agent loop**: `chat → extractToolCalls → runTools → repeat` z `MAX_STEPS`
- **MCP + Native tools**: pliki przez MCP (stdio transport), specjalistyczne operacje jako native tools z handlerami
- **Narzędzie `understand_image`**: czyta plik z dysku → Base64 → wysyła do Vision API z pytaniem
- Agent działa autonomicznie (bez REPL) — jedno query uruchamia cały proces klasyfikacji
- Instrukcja systemowa definiuje zasady rozumowania, nie sekwencję kroków

### `01_04_image_editing` — pętla weryfikacji

- **REPL** z historią konwersacji między turami
- **Narzędzie `analyze_image`** ze strukturyzowanym werdyktem (`accept`/`retry`, `blocking_issues`, `next_prompt_hints`)
- **Style guide** jako plik czytany przez agenta przed generowaniem (augmented function call)
- Parsowanie raportu analizy z tagów tekstowych:

```javascript
const rawVerdict = extractTaggedValue(analysis, "VERDICT").toUpperCase();
return {
  verdict: rawVerdict === "RETRY" ? "retry" : "accept",
  blockingIssues: extractBulletSection(analysis, "BLOCKING_ISSUES"),
  nextPromptHints: extractBulletSection(analysis, "NEXT_PROMPT_HINT")
};
```

### `01_04_json_image` — szablony jako prompt

- **Template cloning workflow**: `COPY → EDIT subject only → READ full JSON → PASS to tool`
- Dwa szablony (`template.json` — szkic, `character-template.json` — cinematic) z locked sections (style, palette, lighting, constraints, negative prompt)
- Agent edytuje wyłącznie sekcję `subject.main` i `subject.details`

### `01_04_image_guidance` — referencje wizualne

- **Mandatory reference** — agent odmawia generowania bez grafiki referencyjnej
- **Inferencja pozy** z opisu użytkownika → dopasowanie do pliku w `workspace/reference/`
- Najczystsza struktura kodu — każde narzędzie w osobnym folderze (`definition.js`, `handler.js`, backend API)
- Shared utilities wydzielone do `native/shared/`

### `01_04_reports` — kompozycja narzędzi w dokument

- **Trzecie narzędzie `html_to_pdf`** (Puppeteer): HTML → PDF z opcjami (format, orientacja, marginesy)
- Szablon HTML z embedded CSS (dark theme, Google Fonts, komponenty: metryki, statusy, tagi)
- Agent klonuje szablon HTML, modyfikuje treść, osadza obrazy (lokalne ścieżki), konwertuje

### `01_04_audio` — nowa modalność

- **Cztery narzędzia** zamiast jednego ogólnego: `transcribe_audio`, `analyze_audio`, `query_audio`, `generate_audio`
- Resumable upload dla plików >20MB (Gemini Files API)
- Multi-speaker TTS — agent przypisuje głosy do rozmówców
- 30 dostępnych głosów z kontrolą stylu przez natural language ("Say cheerfully:", "In a whisper:")

### `01_04_video` — analiza wideo

- YouTube URL jako bezpośrednie źródło danych (bez pobierania)
- **Cztery typy ekstrakcji**: scenes, keyframes, objects, text — każdy z własnym schematem JSON
- Parametry kontroli kosztów: `start_time`/`end_time` (clipping), `fps` (mniej klatek)
- `video_metadata` przekazywane do Gemini API do sterowania przetwarzaniem

### `01_04_video_generation` — kulminacja

- **Pipeline łączący wszystko**: szablon JSON → generowanie klatek start/end → animacja (Kling via Replicate)
- **Polling asynchronicznej operacji** — generowanie wideo trwa minuty:

```javascript
for (let i = 0; i < maxAttempts; i++) {
  const prediction = await replicate.predictions.get(predictionId);
  if (prediction.status === "succeeded") return prediction;
  if (prediction.status === "failed") throw new Error(prediction.error);
  await new Promise(r => setTimeout(r, 5000));
}
```

- Agent sam decyduje czy klatki start/end wymagają poprawek przed animacją
- Mechanizm ciągłości: ostatnia klatka filmu → pierwsza klatka następnego segmentu
